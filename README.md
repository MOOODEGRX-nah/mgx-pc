# MGX(PC)

موقع كتالوج أجهزة كمبيوتر جاهزة (PC Builds) وقطعها — مصمم لتسهيل البحث على
عشاق ومستخدمي الكمبيوتر بالسوق السعودي. بدل ما يضيع الزائر وقته يبحث عن كل
قطعة بمتجر لحالها، يلقى الجهاز كامل بقطعه، روابط الشراء المباشرة، وأسماء
القطع جاهزة للنسخ — كل شي بمكان وحد.

## المميزات

### للزوّار
- **تصفح حسب الفئة**: ثلاث مستويات أداء — High-End / Mid-Range / Budget —
  بأزرار بتصميم زر تشغيل الجهاز (Power Toggle)، تعرض فورًا الأجهزة المطابقة.
- **صفحة تفاصيل كاملة لكل جهاز**: صورة الجهاز ووصفه يظهران بترتيب
  ينعكس تلقائيًا حسب اللغة — الصورة يسار والوصف يمين بالعربي، والعكس بالإنجليزي.
- **قائمة مواصفات تفاعلية**: كل قطعة (الكيس، اللوحة الأم، المعالج، المبرد،
  كرت الشاشة، الرام، وأنواع التخزين الأربعة) بصف قابل للفتح يعرض خيارين:
  - رابط شراء مباشر من المتجر مع شعار المتجر تلقائيًا (Amazon, AliExpress، أو أي متجر تضيفه)
  - اسم القطعة جاهز للنسخ بضغطة وحدة
- **قسم أدوات إضافية** بنفس الفكرة (مفكات، معجون حراري...) لكل جهاز.
- **دعم ثنائي اللغة بالكامل** (عربي/إنجليزي) مع زر تبديل فوري يقلب اتجاه
  الصفحة (RTL/LTR) تلقائيًا.

### للأونر (لوحة التحكم)
- **نظام دخول آمن** بحساب مستقل (بريد إلكتروني + كلمة مرور)، محمي بالكامل
  عن أي زائر آخر.
- **إدارة الأجهزة**: إضافة، تعديل، حذف — مع اسمين (عربي/إنجليزي)، صورة،
  وصف، وتصنيف الفئة.
- **إدارة القطع والأدوات** لكل جهاز بشكل منفصل: النوع، الاسم، المتجر،
  ورابط الشراء.
- **إدارة المتاجر**: شعارات وروابط قابلة للتوسيع — تقدر تضيف أي متجر جديد
  بشعاره الخاص من اللوحة مباشرة، بدون أي تعديل بالكود.

## التصميم

هوية بصرية داكنة بالكامل (أحمر/أسود) مستوحاة من عالم الـ PC Hardware —
أزرار الفئات مصممة كزر تشغيل، وقوائم المواصفات مستوحاة من ملصقات
السبيكشيت الموجودة على علب القطع الحقيقية.

## الستاك التقني

| الطبقة | التقنية |
|---|---|
| الواجهة | React + TypeScript + Vite + Tailwind CSS v4 |
| الترجمة | react-i18next |
| قاعدة البيانات والدخول | Supabase (Postgres + Auth + Storage) |
| الاستضافة | GitHub + Vercel |

## هيكلة المشروع

```
src/
  components/        مكونات عامة (Navbar, BuildCard, SpecRow...)
  components/admin/  مكونات لوحة التحكم (BuildForm, PartsEditor...)
  pages/             الصفحات (Home, BuildDetail, AdminLogin, AdminDashboard)
  context/           AuthContext لإدارة جلسة الدخول
  lib/               عميل Supabase + دوال CRUD
  i18n/              ملفات الترجمة (ar.json / en.json)
  types/             أنواع TypeScript المشتركة
supabase/schema.sql  سكربت إنشاء قاعدة البيانات والحماية
```

## نموذج البيانات

```
stores  →  id, name, logo_url
builds  →  id, name_ar, name_en, category, image_url, description_ar, description_en
parts   →  id, build_id, type, part_name, store_id, store_link
tools   →  id, build_id, tool_name, store_id, store_link
```

كل القطع والأدوات مرتبطة بجهاز واحد (`build_id`)، وكل قطعة/أداة ممكن تكون
مرتبطة بمتجر (`store_id`) لعرض شعاره تلقائيًا.

## الأمان

- الزوّار يقدرون فقط "يقرؤون" البيانات.
- فقط حساب الأونر المسجّل دخوله عبر Supabase Auth يقدر يضيف/يعدّل/يحذف
  أي شي — مفروض على مستوى قاعدة البيانات نفسها (Row Level Security)،
  مو بس بالواجهة.
- صفحة `/admin` محمية بالكامل وتحوّل أي زائر غير مسجّل دخول مباشرة لصفحة الدخول.
